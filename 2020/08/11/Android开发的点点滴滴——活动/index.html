获取EditText中输入的内容

public class Main extends AppCompatActivity implements View.OnClickListener{
    private EditText editText;

    @Override
    protected void onCreate(Bundle savedInstanceState){
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        Button button = (Button) findViewById(R.id.button);
        editText = (EditText) findViewById(R.id.edit_text);
        button.setOnClickListener(this);
    }

    @Override
    public void onClick(View view){
        switch(view.getId()){
            case R.id.button
                String inputText = editText.getText().toString();
                // 使用Toast显示出来
                Toast.makeText(Main.this, inputText,Toast.LENGTH_SHORT).show();
                break;
            default:
                break;
        }
    }
}

使用隐式Intent

<intent-filter>
    <action android:name="com.example.activitytest.ACTION_START" />
    <category android:name="android.intent.category.DEFAULT" />
    <category android:name="com.example.activity.MY_CATEGORY" />
</intent-filter>
button.setOnClickListener(new View.OnClickListener(){
    @Override
    public void onClick(View view){
        Intent intent = new Intnet("com.example.activity.ACTION_START");

    }
})

返回数据的最佳写法

当你需要启动活动2，但是启动活动2必须要从活动1传递一些必要的数据时，你为了知道传递的是什么数据，就必须翻看活动2的源码，这非常的不方便，可以通过启动方法来使启动活动变的简单

// 修改Second中的代码

生命周期

活动状态

  1. 运行状态
  2. 暂停状态
  3. 停止状态
  4. 销毁状态

活动的生存期

onCreate()：活动第一次被创建的时候调用

onStart()：活动有不可见变为可见的时候调用

onResume()：活动准备好和用户进行交互的时候调用

onPause()：系统准备去启动或者恢复另一个互动的时候调用

onStop()：活动完全不可见的时候调用

onDestroy()：活动被销毁之前调用

onRestart()：活动由停止状态变为运行状态之前调用

  • 完整生存期：从onCreate()到onDestory方法之间所经历的
  • 可见生存期：从onStart()到onStop()方法之间所经历的
  • 前台生存期：从onResume()到onPause()方法之间所经历的
