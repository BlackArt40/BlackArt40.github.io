git 常用命令

发表于 2020-03-29 更新于 2020-08-17 分类于 命令

作为一名git的初学者，有部分git命令时间长了不使用容易忘记，在这里简单的记录一下，详细的教程请参见廖雪峰的git教程

创建版本仓库

git init

将文件添加仓库

git add

将文件提交到仓库

git commit -m "message"

查看仓库当前状态

git status

查看各个版本文件具体修改的内容

git diff

显示从最近到最远的提交日志

git log

觉得输出信息太多

git log --pretty=oneline

版本回退

git reset --hard HEAD^

回到指定版本

git reset --hard commit id

查看历史命令

git reflog

撤销工作区的修改

git checkout --file

将暂存区的修改撤销，重新放到工作区

git reset HEAD <file>

删除文件

git rm

创建ssh

ssh-keygen -t rsa -C "youremail@example.com"

本地仓库推送到远端

git push origin master

第一次推送

git push -u origin master

克隆仓库

git clone 网址

创建分支

git branch <name>

创建+合并分支

git checkout -b <name>

git switch -c <name>

查看分支

git branch

切换分支

git checkout <name>

git switch <name>

分支合并到master上

git merge <name>

不删除分支信息

git merge --no-ff <name>

删除分支

git branch -d <name>

查看分支合并图

git log --graph

储藏分支

git stash

恢复分支

git stash apply + git stash drop

git stash pop

复制特定的提交到分支

git cherry-pick <commit>

查看远程库的信息

git remote

or

git remote -v

推送分支

git push origin branch-name

从远端抓取分支

git pull

在本地创建和远程分支对应的分支

git checkout -b branch-name origin/branch-name

建立本地分支和远程分支的关联

git branch --set-upstream branch-name origin/branch-name

创建标签

git tag <name>

查看所有标签

git tag

查看标签信息

git show <tagname>

推送标签

git push origin <tagname>

git push origin --tags

删除标签

git tag -d <tagname>

git push origin :refs/tags/<tagname>
