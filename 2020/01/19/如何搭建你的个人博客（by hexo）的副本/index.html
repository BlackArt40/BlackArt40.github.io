如何搭建你的个人博客（by hexo）

Posted on 2020-01-19 Edited on 2020-07-25 In 个人博客

什么是Hexo？

Hexo是一个快速、简洁且高效的博客框架。Hexo使用Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用自选的主题生成静态网页

配置基础

安装node.js

Node.js为大多数平台提供了官方的安装程序。node.js中文网

其他安装方法：

  • Windows：通过nvs或者nvm安装。
  • Mac：使用Homebrew或MacPorts安装。
  • Linux：使用NodeSource(https://github.com/nodesource/distributions)安装。
Windows用户

使用Node.js官方安装程序时，请确保勾选**ADD TO PATH**选项（默认已勾选）
For Mac/Linux用户

如果在尝试安装Hexo的过程中出现*EACCES*权限错误，请遵循由[npmjs发布的指导](https://docs.npmjs.com/resolving-eacces-permissions-errors-when-installing-packages-globally)修复该问题。强烈建议**不要**使用root、sudo等方法覆盖权限

通过cnpm安装Hexo

  1. 通过node -v 和npm -v检测当前版本

  2. 利用npm安装cnpm（npm国内下载速度不太行，选用淘宝镜像）

    npm install -g cnpm --registry=http://registry.npm.taobao.org

  3. 利用cnpm安装hexo

    cnpm install -g hexo-cli
(自动下载)

  4. 建立一个空的文件夹放置网页主体

    mkdir blog

这步完成之后你可以进入C盘，点开Windows文件夹中的System32文件夹找到blog文件夹，后续操作还是在cmd中进行

  1. 进入blog
    cd blog
  2. 使用hexo生成博客
    hexo init
（它会自动生成）
    如果出现git报错，要去git官网下载git进行安装git。

在本地使用个人博客

启动hexo

hexo s

它会为你显示一个网址，在浏览器输入进入，就是hexo为你生成的网址

创建新的博客文章

hexo n “我的第一篇博客文章”

自动生成该文件

修改博客文章中的内容

进入blog/source/_posts可以查看刚刚新建的文档

可以通过vs code 软件编辑该文档

将文章部署到网页中

hexo g

Hexo文件夹中各文件信息

_config.yml

网站的配置信息，你可以在此配置大部分的参数。

package.json

应用程序的信息。EJS,Stylus和Markdown renderer已默认安装，你可以自由移除

package.json
{
  "name": "hexo-site",
  "version": "0.0.0",
  "private": true,
  "hexo": {
    "version": ""
  },
  "dependencies": {
    "hexo": "^3.8.0",
    "hexo-generator-archive": "^0.1.5",
    "hexo-generator-category": "^0.1.3",
    "hexo-generator-index": "^0.2.1",
    "hexo-generator-tag": "^0.2.0",
    "hexo-renderer-ejs": "^0.3.1",
    "hexo-renderer-stylus": "^0.3.3",
    "hexo-renderer-marked": "^0.3.2",
    "hexo-server": "^0.3.3"
  }
}

scafflods

模板文件夹。当你新建文章时，Hexo会根据scaffold来建立文件。

Hexo的模板是指在新建的文章文件中默认填充的内容。例如，如果你修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。

source

资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes

主题文件夹。Hexo 会根据主题来生成静态页面。

更换主题

下载主题相关文件

这里推荐两个网站，你也可以去GitHub里找自己喜欢的主题进行下载
halo
yilia

git clone https://github.com/halo-dev/halo.git themes/halo
git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

通过这两行代码可以下载halo和yilia两个主题文件到blog/themes中

配置主题文件

在blog文件夹中找到_config.yml文件

用vs code将它打开

将其改成theme: halo

记得:后要加一个空格

配置主题成功后

在cmd中运行下面这段代码

hexo clean
hexo g
hexo s

进入local 4000 查看相关主题

如何修改主题文件，在GitHub中已有现成的方法，在这里不做过多说明(去阅读README文件)

将配置好的网站推到GitHub中

  • 创建GitHub仓库

进入自己的GitHub网站

新建一个仓库(new repository)

一定是你的昵称.github.io

(用户部署个人博客的GitHub仓库的命名必须符合特定要求才行)

然后create，就完成了

  • 在cmd中的操作

在blog文件夹中安装相关插件

cnpm install --save hexo-deployer-git
  • 设置config文件

设置blog文件中的_config.yml文件(用vs code 打开)

在最底部有一个Deployment配置

deploy:
 type: git
 repo: 刚刚创建的仓库的地址(ssh)
 branch: master
  • 部署到远端
    hexo d

部署到远端出现错误的解决办法

  1. 打开Git Bash 查看电脑上是否已经存在SSH密钥

    cd ~/.ssh
sh.exe": cd: /c/Users/Lin/.ssh: No shch file or directory

    若出现上面的显示无法找到该文件则要创建新的ssh key

  2. 创建新的ssh key

ssh -keygen -t rsa -C"your_email@youremail.com"

执行这条命令会上图体是文件保存路径，可以直接按Enter

然后提示输入 passphrase(密码)，输入两次(可以不输直接两次Enter)

你输入的密码你是看不见的

然后会在.ssh目录生产两个文件：id_ras和_rsa.pub

用记事本打开.ssh目录下的id_rsa.pub文件，复制里卖弄的内容(这是你生成的电脑的公钥)

id_rsa文件里面是你自己的私钥，不要交给别人

  1. 复制ssh key到GitHub

这一步是到GitHub的，如果自己有搭建git服务器，则可以直接去服务器上(CentOS)将id_rsa.pub内的公告密钥导入到/home/git/.ssh/authorized_keys文件里，一行一个）参考网址：廖雪峰的官方网站-搭建git服务器

On the Github site Click “Account Settings” > Click “SSH Keys” > Click “ADD SSH Key”

打开GitHub网站，点击右上角扳手图标，然后点击左边菜单的ssh key，然后右边页面的 add ssh key，将复制的内容粘贴到GitHub的key中，title可以不填，直接保存即可

  1. 测试ssh链接到GitHub
ssh -T git@github.com

出现successfully就ok

  1. 设置自己的git信息
git config --global user.name"Firstname Lastname"(此处name可修改为不是用于登陆GitHub的登陆名)
git config --global user.email"your_email@youremail.com"

设置自己的git信息即完成安装和设置，可以输入git config –list查看自己的git信息

  1. 重新部署
hexo d
