前言

什么是Shell

对于内核来说Shell是Linux/Unix的一个外壳,它负责外界与Linux内核的交互,接受用户或其他应用程序的命令,然后把这些命令转化成内核能理解的语言,传给内核,内核是真正干活的,干完之后再把结果返回到用户或者应用程序.

简单的来说,Shell就是命令行.

Shell的分类

Linux/Unix提供了很多种Shell，不同的shell具备不同的功能，shell还决定了脚本中函数的语法，Linux中默认的shell是/bin/bash

想知道你的系统有几种shell，可以通过以下命令查看：

显示如下:

bash这个是目前大多数Linux系统默认使用的shell，全名是BourneAgain Shell，一共有40个命令。包含的功能几乎可以涵盖shell所具有的功能，所以一般的shell脚本都会指定它为执行路径。

在 Linux 里执行这个命令和 Mac 略有不同， Mac 里多了一个 zsh，也就是说 OS X 系统预装了个 zsh(最新版本默认).

什么是Zsh

  • Oh My Zsh是一个开源的由社区驱动的用于管理zsh配置的框架

  • 包含了大量的插件和主题样式,能极大的提升使用shell终端的舒适度

  • Zsh兼容Bash,据传说99%的Bash操作和Zsh是相同的

  • 默认CentOS/Ubuntu/MacOS系统使用的是Bash,不过新版MacOS使用的是Zsh

Zsh安装

如果你用 Mac，就可以直接看下一节，Mac默认已经安装；
如果你用 Redhat Linux，执行：sudo yum install zsh；
如果你用 Ubuntu Linux，执行：sudo apt-get install zsh；

Ubuntu下git的安装和使用

安装git

  1. 在终端中输入git可检查是否已经安装，若没安装可以根据提示的命令行进行安装

    $ git

    ps:很多软件都可以通过这种方式安装。

  2. 根据提示命令进行安装

    $ sudo apt install git

  3. 可能出现（没有可用的软件包git，但是它被其他的软件包引用了）错误

    这可能意味着这个缺失的软件包可能已被废弃，或者只能在其他发布源中找到

  4. 此时只需要更新一下源

    $ sudo apt-get update

  5. 待源更新完毕之后再次输入第二步的安装命令

    $ sudo apt install git

  6. 等待安装完毕之后再次输入git，就会发现已经成功安装

  1. 安装完成后，进行最后一步设置，在命令行输入：

    $ git config --global user.name "Your Name"

    $ git config --global user.email "email@example.com"

如何搭建你的个人博客（by hexo）

什么是Hexo？

Hexo是一个快速、简洁且高效的博客框架。Hexo使用Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用自选的主题生成静态网页

配置基础

安装node.js

Node.js为大多数平台提供了官方的安装程序。node.js中文网

其他安装方法：

  • Windows：通过nvs或者nvm安装。
  • Mac：使用Homebrew或MacPorts安装。
  • Linux：使用NodeSource(https://github.com/nodesource/distributions)安装。

通过cnpm安装Hexo

  1. 通过node -v 和npm -v检测当前版本

    这步完成之后你可以进入C盘，点开Windows文件夹中的System32文件夹找到blog文件夹，后续操作还是在cmd中进行

  5. 进入blog

    如果出现git报错，要去git官网下载git进行安装git。

Hexo文件夹中各文件信息

_config.yml

网站的配置信息，你可以在此配置大部分的参数。

package.json

应用程序的信息。EJS,Stylus和Markdown renderer已默认安装，你可以自由移除

scafflods

模板文件夹。当你新建文章时，Hexo会根据scaffold来建立文件。

Hexo的模板是指在新建的文章文件中默认填充的内容。例如，如果你修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。

source

资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes

主题文件夹。Hexo 会根据主题来生成静态页面。
