Ubuntu换源

发表于 2020-03-28 更新于 2020-08-11 分类于 Ubuntu

  1. 登陆root用户
    su

  2. 先备份原来的源列表文件

    mv /etc/apt/sources.list /etc/apt/sourses.list.backup

  1. 新建源列表文件

    vim /etc/apt/sources.list

  2. 加入源地址（推荐用阿里云源）

deb http://mirrors.aliyun.com/ubuntu/ bionic main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-security main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-updates main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-proposed main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ bionic-backports main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic-security main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic-updates main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic-proposed main restricted universe multiverse
deb-src http://mirrors.aliyun.com/ubuntu/ bionic-backports main restricted universe multiverse

推荐将该网站的网址输入到Ubuntu的浏览器中，然后用鼠标左键选中源地址，在vim界面按鼠标中键就可以快捷复制了

  1. 更新软件列表

    sudo apt-get update

  2. 更新软件（把已安装的软件与刚刷新的软件列表中的版本进行对比，如果发现已安装的软件版本太低，就会提示更新）

    sudo apt-get upgrade

如何在markdown中添加图片

发表于 2020-03-27 更新于 2020-08-11 分类于 文本编辑

插图最基础的格式是：

  • [alt text](图片链接”optional title”)

Alt text:它是图片的ALT标签，用来描述图片的关键字，可以不写。（直接显示在图片下面）

图片链接：可以是图片的本地地址或者是网址

“optional title”：鼠标悬置在图片上会出现的文字，可以不写

插入本地图片

在基础语法的口号中填入图片的位置路径即可，支持绝对路径和相对路径。

例如：

![avatar](/source/picture/1.png)

缺点：不灵活不好分享，本地图片的路径更改或丢失都会造成MarkDown文件调不出图片。（不推荐）

插入网络图片

只需要在基础语法的括号中填入图片的网络链接即可。

例如：

![avatar](http://baidu.com/pic/cat.png)

使用图床上传图片

可以在进入图床上传自己的图片，在里面可以自己选择代码格式，很方便。

使用有道云笔记上传照片

我们只需要把图片保存到有道云笔记中

然后选择分享，点击复制链接

在网页中加载出来，然后右键点击图片复制图片地址

就可以显示出来了。是不是也很方便。

将图片存入markdown文件

用base64转码工具把图片转成一段字符串，然后把字符串填到基础格式中链接的那个位置。

例如：

![avatar][base64str]

不是很推荐这种方法，因为这一段字符串一般很长，很影响阅读体验。

Ubuntu下git的安装和使用

发表于 2020-03-27 更新于 2020-08-11 分类于 Ubuntu

安装git

  1. 在终端中输入git可检查是否已经安装，若没安装可以根据提示的命令行进行安装

    $ git

    ps:很多软件都可以通过这种方式安装。

  1. 根据提示命令进行安装

    $ sudo apt install git

  2. 可能出现（没有可用的软件包git，但是它被其他的软件包引用了）错误

    这可能意味着这个缺失的软件包可能已被废弃，或者只能在其他发布源中找到

  3. 此时只需要更新一下源

    $ sudo apt-get update

  4. 待源更新完毕之后再次输入第二步的安装命令

    $ sudo apt install git

  5. 等待安装完毕之后再次输入git，就会发现已经成功安装

  1. 安装完成后，进行最后一步设置，在命令行输入：

    $ git config --global user.name "Your Name"

    $ git config --global user.email "email@example.com"

MarkDown基本语法

发表于 2020-01-21 更新于 2020-08-11 分类于 文本编辑

MarkDown是一种纯文本格式的标记语言。通过简单的标记语法，它可以使普通文本内容具有一定的格式。
在我写博客时的文件为.md文件，为了防止忘记部分语法，在此介绍

  • 优点：

1.因为是纯文本，所以只要支持MarkDown的地方都能获得一样的编辑效果，可以让作者摆脱排版的困扰，专心写博客
2.操作简单

  • 缺点：

1.需要记一些语法
2.有些平台不支持MarkDown边集模式

一、标题

在想要设置标题的文字前面加#来表示
一个#是一级标题，两个#是二级标题，以此类推。最高支持六级标题。
注：标准语法一般在#后跟一个空格再写文字

示例：

# 这是一级标题
## 这是二级标题
### 这是三级标题
#### 这是四级标题
##### 这是五级标题
###### 这是六级标题

二、字体

  • 加粗

要加粗的文字左右分别用两个*包起来

  • 斜体

要倾斜的文字左右分别用一个*包起来

  • 斜体加粗

要倾斜和加粗的文字左右分别用三个*包起来

  • 删除线

要加删除线的文字左右用两个~~包起来

  • 下划线

下划线可以通过HTML的标签来实现

  • 脚注

脚注时对文本的补充说明。

示例：

**这是加粗的文字**
*这是倾斜的文字*
***这是斜体加粗的文字***
~~这是加删除线的文字~~
<u>这是带下划线的文字</u>
这是要注明的文字[^1]
[^1]:1111

效果如下：

这是加粗的文字
这是倾斜的文字
这是斜体加粗的文字
这是加删除线的文字
这是带下划线的文字
创建脚注格式类似这样[^脚注]
[^脚注]
[^脚注]:这是要标注的文字。

三、引用

在引用的文字前加>即可。引用也可以嵌套，如加两个>>三个>>>
貌似是可以一直加下去的

示例：

>这是引用的内容
>>这是引用的内容
>>>>>>这是引用的内容

效果如下：

这是引用的内容

这是引用的内容

这是引用的内容

八、表格

语法：

表头|表头|表头
--|：-- ：|--:
内容|内容|内容
内容|内容|内容

第二行分割表头和内容。
- 有一个就行
文字默认居左
-两边加：表示文字居中
-右边加：表示文字居右
注：原生的语法两边都要用|包起来。此处省略

示例：

姓名|学号|年级
–|：–：|–：
一|10|大学
二|20|大学
三|20|大学

九、代码

语法：
单行代码：代码之间分别用一个反引号包起来

`代码内容`

代码块：代码之间分别用三个反引号包起来，且两边的反引号单独占一行

(```)

代码、、、
代码、、、
代码、、、

(```)

注：为了防止转译，前后三个反引号处加了小括号，实际时没有的，这里只是用来演示，实际中去掉两边小括号即可。

示例：

单行代码

System.out.print

代码块

for(int i = 0; i < 10; i++)
{
    sum += i;
}
